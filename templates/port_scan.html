扫描配置
支持范围格式（如：1-1000）或逗号分隔（如：80,443,8080）
建议值：0.5-2 秒，越小扫描越快但可能漏掉慢响应端口
建议值：100-300，越大扫描越快但占用资源更多
0 表示不限速，设置 1 表示 1 包/秒（用于绕过防火墙速率限制）
0 表示无延迟，设置 1000 表示每端口间隔 1 秒（慢速扫描）
TCP SYN 扫描：发送 SYN 包，收到 SYN+ACK 表示开放，收到 RST 表示关闭，无响应表示过滤（快速，需要代理程序）
TCP FIN 扫描：发送 FIN 包，收到 RST 表示关闭，无响应表示开放/过滤（隐蔽，需要代理程序）
TCP Null 扫描：发送无标志位的包，收到 RST 表示关闭，无响应表示开放/过滤（隐蔽）
TCP Xmas 扫描：发送 FIN+PSH+URG 包，收到 RST 表示关闭，无响应表示开放/过滤（隐蔽，像圣诞树一样）
TCP ACK 扫描：发送 ACK 包，收到 RST 表示未被过滤，无响应表示被防火墙过滤（用于探测防火墙规则）
TCP FIN+SYN 等异常扫描：发送异常的标志位组合，用于测试防火墙对异常包的检测规则
TCP 连接扫描：通过建立完整 TCP 连接检测端口（最准确，无需代理程序）
